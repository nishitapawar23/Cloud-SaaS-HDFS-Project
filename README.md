# Cloud SaaS using HDFS

## Description
This project implements a basic Software as a Service (SaaS) cloud system over a Local Area Network (LAN) using Hadoop Distributed File System (HDFS). It simulates cloud storage operations like file upload and download with basic security concepts.

## Features
- File upload and download using HDFS
- File storage in distributed blocks
- Simulated cloud controller functionality
- Basic encryption concept for secure data handling

## Technologies Used
- HDFS (Hadoop Distributed File System)
- Linux
- Command Line Interface

## Workflow
1. Start Hadoop services
2. Create directory in HDFS
3. Upload file to HDFS
4. Store file in distributed blocks
5. Retrieve file from HDFS

## Architecture
This project simulates a cloud environment over a LAN where files are divided into blocks and stored across distributed nodes using HDFS.

## Key Concepts
- Distributed Storage (HDFS)
- Data Block Management
- Fault Tolerance
- Basic Security (Encryption concept)

## Commands Used
Refer to the `commands.txt` file for all HDFS commands used in this project.
